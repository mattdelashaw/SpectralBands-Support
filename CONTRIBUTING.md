# Contributing to SpectralBands Support

Thank you for your interest in improving **SpectralBands**! While the core application source code is private, we value community input to improve documentation, track bugs, and refine our feature roadmap.

## 📡 How You Can Help

### 1. Reporting Bugs
- **Search first:** Check [existing issues](https://github.com/mattdelashaw/SpectralBands-Support/issues) to see if the problem has been reported.
- **Be detailed:** Use our [Bug Report Template](.github/ISSUE_TEMPLATE/bug_report.yml). Include your iOS version, SDR hardware, and the server protocol (`rtl_tcp` or `WebSDR`).
- **Provide logs (if possible):** If you see a specific error message in the "Uplink Config" status area, please include it.

### 2. Suggesting Features
- **Think SDR-first:** We prioritize features that improve the core RF monitoring and demodulation experience.
- **Use the template:** Fill out the [Feature Request Template](.github/ISSUE_TEMPLATE/feature_request.yml) with a clear use case.

### 3. Improving Documentation
- If you have tips for setting up `rtl_tcp` on specific platforms (Raspberry Pi, OpenWRT, etc.) or hosting a `WebSDR` server, we welcome pull requests to improve the `README.md` or dedicated guides.

## 📜 Repository Guidelines
- **Be respectful:** We follow a [Code of Conduct](CODE_OF_CONDUCT.md).
- **No sensitive data:** Never post IP addresses, ports, or credentials for private servers in public issues.
- **Focus on Support:** This repo is for support only. Unrelated discussions or spam will be closed.
